Why Self-hosted OSRM? Public OSRM is rate-limited and slow for bulk queries. With local OSRM, you can process 1.2 lakh orders in 2-3 minutes instead of hours. One-time setup takes about 20 minutes.

1 Install Homebrew (if not installed) 2 min

Open Terminal (Cmd + Space, type "Terminal") and paste:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

If you already have Homebrew, skip this step. Check by running brew --version.

2 Install OSRM Backend 3 min

brew install osrm-backend

Verify installation: osrm-extract --version should show v6.0.0 or similar.

3 Download India Map 7-10 min

Create a folder and download the India OpenStreetMap data (~1.6 GB):

mkdir -p ~/osrm-data && cd ~/osrm-data curl -L -o india-latest.osm.pbf https://download.geofabrik.de/asia/india-latest.osm.pbf

This downloads the full India road network. Download time depends on your internet speed.

4 Process the Map 15-25 min

Run these three commands one at a time (each takes a few minutes). Wait for each to finish before running the next:

# Step 4a: Extract road data (~5 min) cd ~/osrm-data osrm-extract -p /opt/homebrew/opt/osrm-backend/share/osrm/profiles/car.lua india-latest.osm.pbf

Wait until you see "extraction finished" and the terminal prompt (%) returns.

# Step 4b: Partition the graph (~8 min) osrm-partition india-latest.osrm

Wait until you see "Bisection took X seconds" and the prompt returns.

# Step 4c: Customize weights (~1 min) osrm-customize india-latest.osrm

Wait until you see "Graph writing took X seconds" and the prompt returns.

5 Start the OSRM Server 10 sec

cd ~/osrm-data osrm-routed --algorithm mld india-latest.osrm --port 5000

You should see "running and waiting for requests". Keep this terminal open!

6 Test It 10 sec

Open a new terminal tab (Cmd + T) and run:

curl "http://localhost:5000/route/v1/driving/77.5946,12.9716;77.6200,13.0000?overview=false"

You should see a JSON response with "code":"Ok". If yes, you're all set!

Now switch to the Upload & Calculate tab, select "Self-hosted OSRM", and start analyzing.

💡 Tips for Daily Use

You only need to do Steps 1-4 once. After that, just run Step 5 each time you want to use the tool:

cd ~/osrm-data && osrm-routed --algorithm mld india-latest.osrm --port 5000

To update the map data (recommended monthly), re-download and re-process Steps 3-4.

1 Install Docker Desktop 5 min

Download and install Docker Desktop from docker.com/products/docker-desktop.

After installation, restart your computer and ensure Docker Desktop is running (check the system tray icon).

2 Download India Map 7-10 min

Open PowerShell and run:

mkdir C:\osrm-data cd C:\osrm-data Invoke-WebRequest -Uri "https://download.geofabrik.de/asia/india-latest.osm.pbf" -OutFile "india-latest.osm.pbf"

3 Process & Run OSRM via Docker 20-30 min

Run these commands one at a time in PowerShell:

# Extract (~5 min) docker run -t -v C:\osrm-data:/data ghcr.io/project-osrm/osrm-backend osrm-extract -p /opt/car.lua /data/india-latest.osm.pbf
# Partition (~8 min) docker run -t -v C:\osrm-data:/data ghcr.io/project-osrm/osrm-backend osrm-partition /data/india-latest.osrm
# Customize (~1 min) docker run -t -v C:\osrm-data:/data ghcr.io/project-osrm/osrm-backend osrm-customize /data/india-latest.osrm

4 Start the Server 10 sec

docker run -t -p 5000:5000 -v C:\osrm-data:/data ghcr.io/project-osrm/osrm-backend osrm-routed --algorithm mld /data/india-latest.osrm --port 5000

Keep this terminal open. Test at http://localhost:5000.

👥 Sharing OSRM with Your Team

If one person sets up OSRM, the whole team can use it! The person running the server just needs to share their IP address. Others enter that IP in the OSRM URL field (e.g., http://192.168.1.100:5000).

To find your IP, run: ipconfig getifaddr en0 (macOS) or ipconfig (Windows) or hostname -I (Linux).
