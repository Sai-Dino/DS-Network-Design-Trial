# benchmark_103_store_metadata.csv

## Source

Built from `bangalore_store_sizes.csv` (authoritative store sizes in sq ft)
matched to the 103 fixed stores in `Store details - 103 old stores.csv`.

Generated by: `scripts/build_benchmark_103_metadata.py`

## Matching logic (93 of 103 matched)

Three strategies applied in priority order:

1. **Direct** (63 stores) — lowercase `site_ID` matched to lowercase `SiteID`
2. **Crosswalk** (27 stores) — `analysis/old_103_to_151_site_crosswalk.csv`
   maps old site IDs to new IDs that appear in the sizes file
3. **Store-code prefix** (3 stores) — e.g. `ben_042` matched via
   `ben_042_wh_hl_01` in the `Store code` column

10 stores had no size data and default to `super_eligible_fixed = 0`:
`ben_155, ben_166, ben_192, ben_193, ben_216, ben_225, ben_226, ben_232, ben_235, ben_243`

## Super-eligibility threshold

`super_eligible_fixed = 1` when `store_sqft > threshold`, else `0`.

Default threshold: **4500 sq ft** (from `DEFAULT_FIXED_SUPER_MIN_SQFT`
in `network-optimizer/server.py` line 137).

Configurable at three levels (highest precedence first):

| Level | Mechanism |
|-------|-----------|
| CLI | `--threshold 5000` on `scripts/build_benchmark_103_metadata.py` |
| Environment | `BENCHMARK_103_SUPER_MIN_SQFT=5000` |
| Server constant | `DEFAULT_FIXED_SUPER_MIN_SQFT` in `network-optimizer/server.py` |

At runtime the server reads `fixed_super_min_sqft` from API request params,
also defaulting to `DEFAULT_FIXED_SUPER_MIN_SQFT`.

## Rebuild

```bash
python3 scripts/build_benchmark_103_metadata.py
python3 scripts/build_benchmark_103_metadata.py --threshold 5000
```
